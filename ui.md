# 拾猫记 App UI 设计文档 (V1.0 - 最终修正版)
 
## 一、UI 设计总览
 
### 1.1 核心视觉风格
- **主风格**：日系手账风（Japanese Planner/Journal Style）
- **情感传达**：温暖、治愈、安静、陪伴、生活感
- **元素特征**：
    *   **柔和配色**：低饱和度、温馨的色彩搭配。
    *   **手写感**：标题、部分提示文字可模拟手写字体效果。
    *   **纸张纹理**：背景、卡片、弹窗可加入微弱的纸张、布艺、木质纹理，增加手账感和家居感。
    *   **轻拟物**：UI 元素（按钮、图标）在扁平化基础上略带投影、高光，模拟贴纸或实体物品的质感，但避免过于复杂。
    *   **圆角与柔和线条**：大量使用圆角，线条避免锐利。
    *   **卡通插画**：猫咪、道具、装饰等使用日系治愈系卡通插画风格。
 
### 1.2 整体布局原则
- **清晰直观**：功能入口明显，布局不拥挤。
- **视觉层级**：通过大小、颜色、阴影区分信息层级。
- **场景化体验**：客厅专注于“专注学习”，卧室专注于“舒缓放松”，通过场景布局强化功能定位。
 
## 二、通用 UI 组件规范
 
### 2.1 按钮
- **主按钮**：圆角矩形，填充主题色，轻微阴影，文字白色。
- **次按钮**：圆角矩形，边框主题色，背景白色或浅色，文字主题色。
- **图标按钮**：圆形或方形，透明背景，内含图标，点击时有反馈动效。
- **禁用状态**：背景色变浅，文字变灰，无交互效果。
- **点击反馈**：按下时轻微缩小，抬起时轻微放大并弹跳，或背景色短暂变深。
 
### 2.2 输入框
- **风格**：圆角矩形，浅色背景，细边框（主题色或中性色），聚焦时边框变色或加粗。
- **占位符**：浅灰色文字。
- **错误提示**：输入框边框变红，下方显示红色提示文字。
 
### 2.3 卡片 / 弹窗
- **风格**：圆角矩形，白色或浅色背景，轻微投影，模拟纸张或木板质感。
- **内容区**：内部布局清晰，信息分块。
- **关闭按钮**：通常在右上角，或通过点击背景半透明区域关闭。
 
### 2.4 顶部导航 / 状态栏
- **首页**：顶部显示代币、好感度、日期等信息。
- **功能页**：顶部左侧返回按钮，中间标题，右侧可能有关闭或更多操作。
- **风格**：简洁，背景透明或与页面主体背景色融合，无明显分隔线。
 
### 2.5 列表项
- **风格**：圆角矩形卡片，或底部有分隔线的简洁样式。
- **交互**：点击时背景色短暂变深。
 
## 三、色彩规范
 
### 3.1 主色板
- **品牌主色 (Primary Brand Color)**：`#FFD7A3` (柔和的橘黄色，温馨治愈，日系感)
- **强调色 (Accent Color)**：`#8EC0C7` (浅蓝色，清新，与主色形成对比)
- **成功色 (Success Color)**：`#A4D8B9` (浅绿色，表示完成、积极)
- **警告色 (Warning Color)**：`#FAD19A` (柔和橘色，提示、警示)
- **错误色 (Error Color)**：`#F7A7A7` (柔和红色，表示错误、失败)
 
### 3.2 中性色板
- **背景主色 (Background Primary)**：`#FBF9F6` (温暖的米白色，模拟纸张基底)
- **背景辅色 (Background Secondary)**：`#F2EFE9` (略深的米灰色，用于分割或强调)
- **文字主色 (Text Primary)**：`#333333` (深灰色，保证可读性)
- **文字辅色 (Text Secondary)**：`#777777` (中灰色，用于次要信息)
- **文字占位符 (Text Placeholder)**：`#AAAAAA` (浅灰色)
- **边框色 (Border Color)**：`#D9D9D9` (浅灰色)
- **阴影色 (Shadow Color)**：`rgba(0, 0, 0, 0.1)` (轻微黑色透明度)
 
## 四、字体规范
 
### 4.1 字体选择
- **主字体**：
    *   **中文**：思源宋体 (Noto Serif CJK SC) 或霞鹜文楷 (LXGW WenKai) - 模拟手写字体的温馨感，但保持高可读性。如果设备不自带，可考虑更通用的系统默认字体（如苹方、思源黑体作为备选）。
    *   **英文/数字**：'Open Sans' 或 'Roboto' (简洁、现代，与中文字体风格协调)。
- **手写体（可选，用于少量标题或插画文字）**：'庞门正道标题体' 或 '站酷酷黑' 等，但需注意版权和兼容性，或使用设计稿中的图片形式。
 
### 4.2 字号与字重
| 用途           | 字号 (px) | 字重 (Weight) | 颜色        |
| :------------- | :-------- | :------------ | :---------- |
| **主标题**     | 24-28     | Bold / Semibold | Text Primary |
| **次级标题**   | 18-20     | Semibold      | Text Primary |
| **正文**       | 14-16     | Regular       | Text Primary |
| **次要信息/辅助** | 12-13     | Regular       | Text Secondary |
| **图标文字**   | 10-12     | Regular       | Text Secondary |
| **按钮文字**   | 15-16     | Semibold      | 随按钮配色 |
 
## 五、图标规范
 
### 5.1 图标风格
- **线条扁平化**：以细线条为主，轻微填充或双色搭配。
- **圆润柔和**：避免尖锐边角，与整体日系风格保持一致。
- **统一性**：所有图标保持设计语言、线条粗细、填充方式的一致。
- **日系元素**：可适当融入小花、树叶、星星等装饰性元素。
 
### 5.2 图标尺寸
- **交互图标 (如返回、菜单、功能按钮)**：24x24 px, 32x32 px
- **列表图标**：16x16 px, 20x20 px
- **状态图标 (如代币、好感)**：20x20 px, 24x24 px
 
## 六、动效规范
 
### 6.1 通用过渡
- **页面切换**：淡入淡出 (Fade-in/out) 或平滑滑动 (Slide)，时长 200-300ms。
- **元素显隐**：渐显/渐隐 (Fade-in/out)，时长 150-250ms。
- **点击反馈**：按钮或可点击区域在按下时轻微缩小 (Scale down 0.98)，抬起时恢复并伴随轻微弹跳 (Scale up 1.02, then back to 1)。
 
### 6.2 特殊动效
- **猫咪帧动画**：
    *   待机动作 (呼吸、眨眼、甩尾)：循环播放，自然衔接。
    *   互动动作 (进食、玩耍、抚摸反馈)：播放一次后回到待机或特定休息状态。
    *   时长：根据动作复杂度和情绪节奏调整，通常 500ms - 1500ms。
- **奖励获得**：代币/好感值增加时，数字跳动效果，并伴随小动画（如金币/爱心从小图标飞出）。
- **完成任务**：任务勾选时，勾选框有轻微弹出效果。

## 七、前端框架与组件选择 (给 Cursor 的建议)
 
为了高效完成课程作业并保证质量，以下是推荐的前端技术栈和组件库：
 
### 7.1 前端框架
- **React Native** (如果你要开发跨平台移动 App，一次开发多端运行)
    *   **优点**：社区活跃，成熟，跨平台效果好，大量第三方库。
    *   **缺点**：学习曲线相对陡峭，打包体积可能较大。
- **Flutter** (如果你更熟悉 Dart 语言，或者追求更原生、更高性能的 UI 体验)
    *   **优点**：性能出色，UI 渲染强大，一次编写多端运行。
    *   **缺点**：Dart 语言生态相对较小，学习曲线陡峭。
- **Vue (结合 Ionic/NativeScript)** (如果你熟悉 Vue，并且课程允许 PWA 或混合 App 方案)
    *   **优点**：Vue 学习成本低，易上手，Ionic/NativeScript 可以方便地包装成 App。
    *   **缺点**：Native 体验可能不如 React Native/Flutter，打包体积可能较大。
 
**建议：考虑到课程作业通常对原生性能要求不高，且学习成本和社区资源，如果无特定偏好，推荐使用 React Native。**
 
### 7.2 UI 组件库
为了快速构建符合日系手账风格的 UI，并且避免从头手写所有组件，推荐使用：
 
-   **React Native**：
    *   **React Native Paper** (推荐)：提供 Material Design 风格的组件，但其高度可定制性允许你覆盖主题，调整圆角、颜色、字体，使其更贴近日系手账风格。它提供了丰富的卡片、按钮、输入框、Modal 等基础组件，能显著提升开发效率。
    *   **React Native Elements**：另一个选择，组件更偏向通用性，定制性也不错。
    *   **自定义组件**：对于手账风的纸张纹理、贴纸效果，可能需要结合 `Image` 和 `View` 的 `border-radius`, `elevation`/`shadow` 属性，或使用 `react-native-svg` 进行定制绘制。
 
-   **Flutter**：
    *   **Material Components for Flutter (内置)**：Flutter 默认的 Material Design 组件库，定制性非常强，可以通过主题 (Theme) 机制轻松修改颜色、字体、形状 (Shape) 等，可以完美贴合日系手账风格。
    *   **Cupertino Widgets (内置)**：如果你需要一些 iOS 风格的组件，也可以混合使用。
    *   **第三方库**：`getwidget`, `flutter_slidable` 等可以补充特定功能。
 
### 7.3 动画库
-   **React Native**：
    *   **`Animated` API (内置)**：适用于简单的平移、缩放、透明度、旋转等动画。
    *   **`react-native-reanimated` (推荐)**：性能更好的动画库，适用于复杂手势和交互动画。
    *   **`lottie-react-native`**：如果猫咪动画是用 Lottie 格式提供，可以直接集成。
    *   **帧动画**：对于简单帧动画，可以通过序列图片切换实现，或用 `react-native-animatable` 等库。
 
-   **Flutter**：
    *   **`Animation` 和 `AnimationController` (内置)**：Flutter 强大的动画系统，几乎可以实现任何你想要的动画。
    *   **`Rive` (原 Flare)**：用于构建复杂矢量动画，可以导出给 Flutter 使用。
    *   **`Lottie` (通过 `lottie_flutter`)**：同样支持 Lottie 动画。
 
### 7.4 本地存储
-   **React Native**：
    *   **`@react-native-async-storage/async-storage`**：非关系型键值对存储，非常适合你的本地数据需求。
-   **Flutter**：
    *   **`shared_preferences`**：轻量级键值对存储。
    *   **`sqflite`**：如果需要更复杂的关系型数据库，可以使用 SQLite。
 
### 7.5 图标库
-   **React Native**：
    *   **`react-native-vector-icons`** (推荐)：支持多种字体图标库，如 Font Awesome, Material Icons, Feather Icons 等。你可以选择其中风格匹配的图标，或者将其作为占位符，后续替换为定制的 SVG 图标。
-   **Flutter**：
    *   **`Icons` class (内置)**：Flutter 提供了大量的 Material Icons。
    *   **`Font Awesome Flutter`**：可以集成 Font Awesome。
    *   **`flutter_svg`**：用于显示 SVG 矢量图，适合你的日系柔和图标风格。
